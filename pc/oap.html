OnionJumoke App Protocol

A simple, lightweight protocol for distributing iOS IPA files over HTTP.

What is OAP?

OAP (OnionJumoke App Protocol) is a static JSON-based protocol for distributing iOS applications. It allows any HTTP server (including GitHub Pages) to serve as an app repository. The client app fetches a manifest file (apps.json) and presents the available apps for download and installation.

How It Works

The protocol is straightforward:

  1. Host a JSON file (apps.json) on any web server
  2. Each app entry in the JSON describes the app's metadata and points to its IPA file
  3. The client app fetches the JSON and renders a list of available apps
  4. Users tap an app to download and install the IPA directly on their device

JSON Format Reference

{
  "apps": [
    {
      "name": "App Name",
      "bundle_id": "com.example.app",
      "version": "1.0",
      "icon": "icons/app.png",
      "description": "A description of the app.",
      "ipa": "ipas/app.ipa",
      "category": "Utilities",
      "size": "100K"
    }
  ]
}
Field Type Description
namestringDisplay name of the app
bundle_idstringReverse-domain bundle identifier
versionstringVersion string (e.g. "1.0")
iconstringRelative path to the app icon PNG
descriptionstringShort description of the app
ipastringRelative path to the IPA file
categorystringApp category (e.g. "Utilities")
sizestringFile size (e.g. "100K")

Setting Up an OAP Server Locally

  1. Create the directory structure: 
    oap/
  apps.json
  icons/
  ipas/
  2. Create your apps.json manifest with your app entries.
  3. Place IPA files in ipas/ and icons in icons/.
  4. Serve the directory with Python's built-in HTTP server: 
    python3 -m http.server 8080
  5. Point your OAP client to http://your-ip:8080/oap/apps.json.

Requirements

  • A web server (Apache, Nginx, Python http.server, GitHub Pages, etc.)
  • IPA files must be signed for the target device or jailbroken
  • Icon files should be PNG format, 144x144px recommended
  • Server must serve .ipa files with MIME type application/octet-stream

Example

The official OAP repository runs on this website. View the manifest at oap/apps.json and browse available apps at oap/.

OAP is fully open and can be implemented by any client that speaks HTTP and understands JSON.
