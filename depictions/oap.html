OAP



OnionJumoke App Protocol

A simple IPA distribution protocol

What is OAP?

OAP (OnionJumoke App Protocol) is a lightweight, static protocol for distributing iOS IPA files. It uses a simple JSON manifest to list available apps, their metadata, and download links. The protocol is designed to work with GitHub Pages or any static file server.

How It Works

1. A server hosts a JSON file (apps.json) containing the app manifest.

2. The client (e.g. the onionjumoke app) fetches the JSON from a configured URL.

3. Each app entry includes: name, bundle ID, version, icon, description, IPA path, category, and size.

4. The client displays the apps and provides download/install functionality.

JSON Format

{
  "apps": [
    {
      "name": "App Name",
      "bundle_id": "com.example.app",
      "version": "1.0",
      "icon": "icons/app.png",
      "description": "Description text.",
      "ipa": "ipas/app.ipa",
      "category": "Utilities",
      "size": "100K"
    }
  ]
}

The JSON is hosted at example.com/oap/apps.json. All paths (icon, ipa) are relative to that URL.

Setting Up an OAP Server Locally

1. Create the directory structure:

oap/
  apps.json
  icons/
  ipas/

2. Create apps.json with your app entries.

3. Place your IPA files in ipas/ and icons in icons/.

4. Serve the directory with any HTTP server:

python3 -m http.server 8080

5. Point your OAP client to http://your-ip:8080/oap/apps.json.

Requirements

- A web server (Apache, Nginx, Python http.server, GitHub Pages, etc.)

- IPA files must be unsigned or correctly signed for the target device

- Icons should be PNG format, 144x144px recommended

- MIME types must be set correctly for .ipa files (application/octet-stream)
